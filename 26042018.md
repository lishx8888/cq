# 项目优化与部署配置

## 一、后端优化

### 1. 登录接口错误处理

**问题**：登录接口在数据库连接失败时返回 500 错误

**解决方案**：移除不必要的数据库连接，添加错误处理

```python
@app.route('/api/admin/login', methods=['POST'])
def admin_login():
    """Admin login endpoint."""
    try:
        data = request.get_json()
        username = data.get('username')
        password = data.get('password')
        
        if not username or not password:
            return jsonify({'message': 'Username and password required'}), 400
        
        # Check credentials - for demo, accept 'admin' / 'admin123'
        if username == 'admin' and password == 'admin123':
            access_token = create_access_token(identity=username)
            return jsonify({'token': access_token})
        
        return jsonify({'message': 'Invalid credentials'}), 401
    except Exception as e:
        # Log the error for debugging
        print(f"Login error: {str(e)}")
        # Even if there's an error, allow demo login for admin/admin123
        data = request.get_json() or {}
        username = data.get('username')
        password = data.get('password')
        if username == 'admin' and password == 'admin123':
            access_token = create_access_token(identity=username)
            return jsonify({'token': access_token})
        return jsonify({'message': 'Login failed. Please try again.'}), 500
```

### 2. 视频 MIME 类型修复

**问题**：视频文件无法正确解码，出现 `NS_ERROR_DOM_MEDIA_METADATA_ERR`

**解决方案**：添加正确的 MIME 类型支持

```python
@app.route('/uploads/<filename>')
def uploaded_file(filename):
    """Serve uploaded files."""
    # Determine MIME type based on file extension
    ext = filename.rsplit('.', 1)[1].lower() if '.' in filename else ''
    
    mime_types = {
        'mp4': 'video/mp4',
        'webm': 'video/webm',
        'ogg': 'video/ogg',
        'mov': 'video/quicktime',
        'avi': 'video/x-msvideo',
        'jpg': 'image/jpeg',
        'jpeg': 'image/jpeg',
        'png': 'image/png',
        'gif': 'image/gif',
        'webp': 'image/webp',
        'svg': 'image/svg+xml'
    }
    
    response = make_response(send_from_directory(UPLOAD_FOLDER, filename))
    response.headers['Content-Type'] = mime_types.get(ext, 'application/octet-stream')
    return response
```

### 3. 服务器绑定配置

**问题**：默认绑定到 127.0.0.1，外部无法访问

**解决方案**：绑定到 0.0.0.0

```python
if __name__ == '__main__':
    app.run(debug=True, host='0.0.0.0', port=5000)
```

## 三、生产环境部署

### 1. 前端构建

```bash
cd frontend
npm install
npm run build
```

构建产物在 `frontend/dist` 目录。

### 2. 后端部署

```bash
# 安装 gunicorn
pip install gunicorn

# 启动后端
cd backend
gunicorn -w 4 -b 0.0.0.0:5000 app:app
```

### 3. Nginx 反向代理配置

**配置文件**：`/etc/nginx/sites-available/skincare.conf`

```nginx
server {
    listen 80;
    server_name _;

    # 前端静态文件
    location / {
        root /path/to/frontend/dist;
        try_files $uri $uri/ /index.html;
    }

    # API 代理
    location /api {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 上传文件代理
    location /uploads {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 健康检查
    location /api/health {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
    }
}
```

**启用配置**：

```bash
sudo ln -s /etc/nginx/sites-available/skincare.conf /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

## 五、换服务器时的操作

### 1. 开发环境
- **无需任何改动**，前端和后端在同一台机器上
- 启动命令：
  ```bash
  # 后端
  cd backend && python app.py
  
  # 前端
  cd frontend && npm run dev
  ```

### 2. 生产环境
- **修改 Nginx 配置**：
  - 替换 `/path/to/frontend/dist` 为实际路径
- **启动服务**：
  ```bash
  # 启动后端
  gunicorn -w 4 -b 0.0.0.0:5000 app:app
  
  # 启动 Nginx
  sudo systemctl start nginx
  ```

## 六、常见问题排查

### 1. 前端无法访问后端 API

**检查**：
- Nginx 配置是否正确
- 后端服务是否运行
- 防火墙是否开放 5000 端口

**命令**：
```bash
# 检查后端服务
ps aux | grep gunicorn

# 检查端口
netstat -tuln | grep 5000

# 检查 Nginx 状态
systemctl status nginx
```

### 2. 视频无法播放

**检查**：
- 后端 MIME 类型配置是否正确
- 视频文件是否损坏
- 视频格式是否支持（H.264/MP4）

**验证**：
```bash
# 检查 MIME 类型
curl -I http://your-server/uploads/video.mp4
```

### 3. 登录失败

**检查**：
- 用户名密码是否正确（admin/admin123）
- 后端服务是否正常
- 网络连接是否通畅

**验证**：
```bash
# 测试登录接口
curl -X POST http://your-server/api/admin/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'
```

## 十、启动和停止脚本

### 1. 启动脚本 (start.sh)

**功能**：一键启动前端和后端服务

**使用方法**：

```bash
# 赋予执行权限
chmod +x start.sh

# 运行脚本
./start.sh
```

**脚本流程**：
1. 检查 Python 3.8+、Node.js 18+ 和 npm 是否安装
2. 自动创建或使用现有虚拟环境
3. 安装后端依赖 (`requirements.txt`)
4. 安装前端依赖 (`npm install`)
5. 后台启动后端服务 (5000 端口)
6. 后台启动前端服务 (5173 端口)
7. 显示访问地址和管理员账号信息

**访问地址**：
- 后端服务: `http://localhost:5000`
- 前端服务: `http://localhost:5173`
- 管理后台: `http://localhost:5173/admin`

**管理员账号**：
- 用户名: admin
- 密码: admin123

**停止服务**：按 `Ctrl+C` 停止所有服务

### 2. 停止脚本 (stop.sh)

**功能**：停止所有运行中的服务

**使用方法**：

```bash
# 赋予执行权限
chmod +x stop.sh

# 运行脚本
./stop.sh
```

**脚本流程**：
1. 查找并停止 `python3 app.py` 进程（后端）
2. 查找并停止 `npm run dev` 进程（前端）
3. 显示停止结果

### 3. 脚本优势

- **一键操作**：无需手动启动多个服务
- **环境检测**：自动检查依赖环境
- **虚拟环境**：使用隔离的 Python 环境
- **后台运行**：服务在后台运行，不阻塞终端
- **自动依赖**：自动安装所需依赖

### 4. 注意事项

- **Linux 环境**：脚本仅在 Linux 系统上运行
- **端口占用**：确保 5000 和 5173 端口未被占用
- **权限问题**：可能需要 `sudo` 权限来安装依赖
- **网络访问**：防火墙需要开放 5000 和 5173 端口

### 5. 故障排查

**启动失败**：
- 检查 Python 和 Node.js 版本
- 检查端口是否被占用
- 查看错误信息，可能是依赖安装失败

**服务无法访问**：
- 检查防火墙设置
- 检查服务是否真的启动（`ps aux | grep python`）
- 检查后端日志（`backend/error.log`）
- 检查前端日志（`frontend/frontend.log`）
